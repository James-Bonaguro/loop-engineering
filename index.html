Briefing · Loop engineering · June 2026

Prompting was the skill. Loop design is the frontier.

The people who build Claude Code and Codex have moved from writing prompts to designing the loops that run the agent. Every stage of that loop is a control surface, which is where governance actually lives.

The signal

Three people, the same conclusion, in one week.

This is not a vendor talking point. It surfaced from the engineers building the tools, independently, and it moved fast.

Boris Cherny

Claude Code · Anthropic

No longer prompts Claude directly. He runs loops that prompt Claude and decide the next step. In his words, his job is to write loops.

Peter Steinberger

Creator, OpenClaw

Stop prompting coding agents one turn at a time. Design the loops that prompt them, and let the system carry the work.

Addy Osmani

Engineering · Google

Wrote the pattern up and named it loop engineering, placing it one level above agent harness engineering.

The tooling is converging. Claude Code and Codex have landed on the same loop primitives, so the shape of a loop is becoming roughly tool agnostic. The pattern outlives any one product.

The anatomy

Six stages. Each one a control surface.

The distinction that matters

A loop is not a cron job.

Rerunning one prompt on a timer is automation we have had for decades. A real loop discovers what to do, checks its own output, holds state across runs, manages its own cost, and knows its limits.

The work did not get easier. The hard part moved from writing the prompt to designing the control system around it. That is why loop design is harder than prompt engineering, not a shortcut past it.

A loop without evaluation and cost discipline is just expensive automation at scale. Governance is not a tax on this. It is the discipline that makes it production grade.

What this means for adoption

Write the standard around the loop, not the prompt.

The control surfaces above are where an enterprise standard should live. Set policy on evaluation, cost ceilings, permissions, retained data, and handoff thresholds. Prompt quality is a craft concern. Loop control is the governance concern, and it is what makes agentic work safe to put near a real process.
